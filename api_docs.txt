Global Conventions
All endpoints follow RESTful principles. Data is exchanged in JSON format unless noted (file uploads use multipart/form-data). Protected endpoints require Authorization: Bearer <token> in the request header.
Authentication
JWT Bearer Token · Authorization: Bearer <access_token>
Token Expiry
Access token: 15 min  |  Refresh token: 7 days
Rate Limiting
100 requests / minute per user  →  429 Too Many Requests on breach
Pagination
List endpoints support ?page=1&limit=20 query parameters
Error Format
{ "success": false, "error": { "code": 400, "message": "..." } }

Authentication & User Management
Registration · Login · Social Auth · Token Management · Password Recovery
1.1  Registration & Account Verification
POST
/auth/register
Register new user
Creates a new user account. Sends an automated email with a verification token. Account remains unverified until /auth/verify-email is called. CAPTCHA token must be obtained from the client-side Google reCAPTCHA widget.
REQUEST BODY
Field
Type
Required
Description / Example
email
string
Yes
Valid email address — must be unique. Example: "user@example.com"
password
string
Yes
Min 8 chars, at least 1 uppercase, 1 number. Example: "StrongPass1!"
display_name
string
Yes
Public display name. Example: "DJ Khaled"
captcha_token
string
Yes
Google reCAPTCHA v2 response token from the frontend widget.
account_type
string
No
"artist" or "listener". Defaults to "listener" if omitted.
RESPONSE EXAMPLE
{
  "success": true,
  "message": "Registration successful. Please check your email to verify your account.",
  "data": {
    "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "email": "user@example.com",
    "display_name": "DJ Khaled",
    "is_verified": false
  }
}
Error codes: 400 Invalid input / missing fields  |  409 Email already registered  |  422 CAPTCHA validation failed
POST
/auth/verify-email
Verify email address
Validates the email verification token sent after registration. Token is single-use and expires after 24 hours.
REQUEST BODY
Field
Type
Required
Description / Example
token
string
Yes
Verification token from the email link. Example: "abc-123-verification-token"
RESPONSE EXAMPLE
{"success": true, "message": "Account verified. You can now log in."}
Error codes: 400 Invalid or malformed token  |  410 Token expired — use /auth/resend-verification
POST
/auth/resend-verification
Resend verification email
Resends a fresh verification email. Rate-limited to 3 requests per hour per email address.
REQUEST BODY
Field
Type
Required
Description / Example
email
string
Yes
The registered email address. Example: "user@example.com"
RESPONSE EXAMPLE
{"success": true, "message": "Verification email resent. Please check your inbox."}
Error codes: 404 Email not found  |  409 Account already verified  |  429 Rate limit exceeded
1.2  Login & Social Authentication
POST
/auth/login
Login with email & password
Authenticates a verified user and issues a JWT access token and refresh token. Access token must be sent as Bearer in all protected requests.
REQUEST BODY
Field
Type
Required
Description / Example
email
string
Yes
Registered email. Example: "user@example.com"
password
string
Yes
Account password. Example: "StrongPass1!"
RESPONSE EXAMPLE
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "def502abc...",
    "token_type": "bearer",
    "expires_in": 900,
    "user": {
      "user_id": "a1b2c3d4-...",
      "display_name": "DJ Khaled",
      "account_type": "artist",
      "is_premium": false
    }
  }
}
Error codes: 400 Missing fields  |  401 Invalid credentials  |  403 Account not verified  |  403 Account suspended
POST
/auth/google
Google OAuth2 social login
Authenticates or auto-registers a user via Google OAuth2. If no account exists for the Google email, one is created with is_verified = true. The google_token is the ID token returned by the Google Sign-In client library.
REQUEST BODY
Field
Type
Required
Description / Example
google_token
string
Yes
Google ID token from the client-side Google Sign-In SDK. Example: "ya29.a0AfH6SM..."
RESPONSE EXAMPLE
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "def502abc...",
    "token_type": "bearer",
    "expires_in": 900,
    "is_new_user": false
  }
}
Error codes: 400 Missing or empty google_token  |  401 Google token invalid or expired  |  503 Google OAuth service unavailable
📝 If is_new_user is true, redirect to a profile completion screen.

1.3  Password Management
POST
/auth/forgot-password
Request password reset
Sends a password reset link to the given email. Always returns 200 regardless of whether the email exists — prevents user enumeration attacks.
REQUEST BODY
Field
Type
Required
Description / Example
email
string
Yes
Email address of the account. Example: "user@example.com"
RESPONSE EXAMPLE
{"success": true, "message": "If an account with that email exists, a reset link has been sent."}
Error codes: 429 Rate limit exceeded — max 3 reset requests per hour
📝 Always returns 200 even if email is not registered. This is intentional to prevent user enumeration.
POST
/auth/reset-password
Reset password with token
Updates the account password using the reset token received by email. Token expires after 1 hour. All active refresh tokens are invalidated upon success.
REQUEST BODY
Field
Type
Required
Description / Example
token
string
Yes
Password reset token from the email link. Example: "reset-token-xyz"
new_password
string
Yes
New password. Min 8 chars, 1 uppercase, 1 number. Example: "NewStrongPass1!"
RESPONSE EXAMPLE
{"success": true, "message": "Password updated successfully. All sessions have been terminated."}
Error codes: 400 Token invalid or malformed  |  400 Password does not meet requirements  |  410 Token expired

2.3  External Social Links
GET
/users/me/social-links
Get social profile links
🔒 JWT
Returns the list of external social platform links configured on the authenticated user's profile.
RESPONSE EXAMPLE
{
  "success": true,
  "data": {
    "social_links": [
      {"platform": "instagram", "url": "https://instagram.com/djkhaled"},
      {"platform": "twitter",   "url": "https://twitter.com/djkhaled"}
    ]
  }
}
Error codes: 401 Unauthorized
PUT
/users/me/social-links
Update social profile links
🔒 JWT
Replaces the full set of external social links. Full replacement — not a merge. Send empty array [] to clear all links. Max 5 links per user.
REQUEST BODY
Field
Type
Required
Description / Example
links
array
Yes
Array of social link objects. Send empty array [] to remove all links.
links[].platform
string
Yes
Platform identifier. Accepted: "instagram", "twitter", "facebook", "youtube", "website"
links[].url
string
Yes
Full URL starting with https://. Example: "https://instagram.com/djkhaled"
RESPONSE EXAMPLE
{
  "success": true,
  "message": "Social links updated successfully.",
  "data": {"social_links": [{"platform": "instagram", "url": "https://instagram.com/djkhaled"}]}
}
Error codes: 400 Invalid platform value  |  400 URL must start with https://  |  400 Max 5 links exceeded  |  401 Unauthorized
📝 This is a full replacement. Always send the complete desired array.

Global Error Reference
All error responses follow this standard structure:
{
  "success": false,
  "error": {
    "code": 401,
    "message": "Authentication token is missing or invalid.",
    "details": null
  }
}
Code
Status
Common Cause
400
Bad Request
Malformed request body, missing required fields, or failed validation
401
Unauthorized
Missing, expired, or invalid JWT access token
402
Payment Required
Card declined or payment failed (Stripe mock)
403
Forbidden
Authenticated but insufficient permissions or not resource owner
404
Not Found
Requested resource does not exist
409
Conflict
Resource already exists (e.g., duplicate email, track already liked)
410
Gone
Token has expired and can no longer be used
413
Payload Too Large
Uploaded file exceeds the maximum allowed size
422
Unprocessable Entity
Request structure valid but semantic validation failed
429
Too Many Requests
Rate limit exceeded for this endpoint
500
Internal Server Error
Unexpected server-side error — report to DevOps
